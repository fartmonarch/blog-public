## JavaScript 事件循环 (Event Loop) 核心总结

### 1. 核心大前提：单线程与非阻塞
JavaScript 是一门**单线程**语言。这意味着它同一时间只能干一件事。为了防止遇到网络请求、定时器等耗时任务导致页面卡死，JS 引入了**非阻塞异步机制**，通过事件循环（Event Loop）来协调代码的执行。

### 2. 执行模型的三大支柱
*   **主线程 (Main Thread / Call Stack)**: 所有同步代码都在这里排队执行。你可以理解为“当下正在干的活儿”。同步任务执行完，主线程才会去任务队列里找活儿。
*   **微任务队列 (Microtask Queue)**: 主要包括 `Promise.then/catch/finally`、`MutationObserver`。它的特权是：**抢代优先级极高**。
*   **宏任务队列 (Macrotask Queue)**: 主要包括 `setTimeout`、`setInterval`、`I/O` 操作及 UI 渲染。它的特点是：**排在最后，进场最晚**。

### 3. 一次循环的完整链路
1.  **同步先行**：主线程首先执行当前脚本中的所有同步代码，直到执行栈空。
2.  **清空微任**：关键的一步！主线程在准备执行下一个宏任务之前，**必须一次性清空**当前所有的微任务队列。如果某个微任务执行时又产生了新的微任务，也要在本次循环中连带清空。
3.  **渲染衔接**：清空微任务后，浏览器可能会进行一次 UI 渲染更新。
4.  **宏任务登场**：从宏任务队列中取出**一个（且仅一个）**任务放入主线程执行。执行完毕后再回到第 2 步，周而复始。

### 4.总结口诀
**“同步代码走到底，微务清空再宏取。”**

在做这类题目时，记住两点杀手锏：
-   `new Promise(executor)` 的 `executor` 内部代码是**同步**的。
-   `async` 函数中，`await` 右侧紧跟的代码是**同步**执行的，但 `await` 下方的代码被视为 `Promise.then` 内部的逻辑，属于**微任务**。
