## LLM(大语言模型)
LLM即Large Language Model 大语言模型 
最早的架构支持是Google的《Attenion is All You Need》中提及到的Transformer
但是最早将其普惠于大众的是Openai于2022年发布的GPT3.5 虽然当时我还是个高三牲，但是也在B站看到有看到up在使用了 超级的震惊
后续的Gemini、Claude、豆包、deepseak等都是LLM
### LLM生成文字的原理
本质是一个**文字接龙**的游戏，
预测流程：
* 接收输入文本。
* 通过内部矩阵运算预测下一个概率最高的词。
* 将生成的词追加到原始输入序列中。
* 重复上述过程，直到生成特殊的结束标识符。
  
比如我问GPT"篮球是一项什么运动"，它会先运算概率生成"特别"，然后把这两点整合"篮球是一项什么运动: 特别",再按概率生成"健康的运动"，最后整合起来发完整"特别健康的运动"
所以LLM回复都是"一个词一个词"跳出来的

## Token 
> 最近我们国家给他起了一个中文名称:词元

&emsp;因为LLM无法直接处理我们的自然语言,就采用了Token这个最基本处理单元,通过Tokenizer(分词器)来拆分文本最小片段一个个Token,再把Token转换为一个个对应的唯一的数字ID
&emsp;还是举个例子形象一些,还是拿上面"篮球是一项什么运动"这句话举例子好了,会被分成"篮球""是一项""什么""运动"会被分成5个Token,再被分成[45642, 122958, 22032, 10555, 106487]5个唯一的数字ID，这样LLM才能理解你说的话
&emsp;数量估算：(有时ai的理解和我们本身的词性理解有些偏差，可以来[openai官方的Tokenizer](https://platform.openai.com/tokenizer)测试一下
* 1个 Token 平均约等于 0.75个英文单词。
* 1个 Token 约等于 1.5至2个汉字。

而且我发现现在越来越先进的LLM Token划分越智能，"篮球是一项什么运动"这句话用GPT-3来划出18个Token,GPT-5.x就是5个了。

## Context(上下文)
> 平时使用ai和我们对话的时候，感觉视乎ai能记住我们在同一个窗口甚至以前说过的话，再根据之前的情况或环境来回答我们。但其实ai是没有记忆能力的，靠的就是这个Context

理解一下Context的概念：模型在单次任务处理中所接收到的信息总和。
Context的作用就是，让LLM每次回答根据我们和ai的先前的对话记录和目前的问题来回复的
Context Window(上下文窗口)：模型能够单次容纳的最大 Token 数量上限。越先进越智能的模型这个Context Window越好，现在的主流模型已经支持到1000k-2000k这个规模了

### RAG(检索增强生成)
* 解决痛点：当资料量（如长篇产品手册）超过上下文窗口或为了控制成本时，使用 RAG 技术。
* 原理：从海量资料中检索出与当前问题最相关的片段，仅将这些片段发送给模型，而非全量输入。

## Prompt(提示词)
>有些人说话就是好让你理解，但是有些人说的就不知所云。

Prompt工程就是把我们的自然语言进行修饰让LLM更好理解。但是随着LLM不断升级迭代，它们自身的理解能力已经越来越强了，提示词工程需求明显下降。
Prompt同时分为两类
* System Prompt (系统提示词)：
设定者：开发者。
作用：定义模型的人设（如“你是一个数学老师”）、做事规则、语言风格及禁忌。用户通常感知不到其存在。
* User Prompt (用户提示词)：
设定者：最终用户。
作用：用户在对话框中输入的具体问题或任务请求。

## Tool(工具)
>直接问LLM帮我查今天的天气，LLM本身是无法直接调用的天气查询的接口和使用实时的数据回答你，需要通过Tool。

Tool允许模型感知并影响外部环境（如查询实时天气、操作文件）
Tool调用的大概流程
该流程涉及四个关键角色：用户、大模型、工具、平台 (传话筒)。
1. 平台发送：将用户问题与可用工具列表（函数定义）发送给模型。
2. 模型决策：分析需求，决定是否调用工具，并生成包含工具名和参数的指令。
3. 平台执行：平台捕获指令，真实执行外部函数，并获取结果。
4. 模型总结：模型拿到工具返回的数据，进行归纳总结，输出最终的人类语言。

## MCP(模型上下文协议)
>以往每个大模型平台（OpenAI, Google, Anthropic）的Tool接入标准不一，导致Tool的开发者需重复开发。

MCP即Model Context Protocol，推出这个一套统一的工具接入规范，使开发者只需按 MCP 标准编写一次工具，即可在所有支持该协议的 AI 平台间通用，类似于现在大部分手机都采用的 Type-C 接口标准。

## Agent(智能体)
>LLM面对复杂的任务，难以多次调用大量Tool

Agent的特征是具有自我规划、自主调用可以持续工作多步骤推理、工具选择、流程选择直到用户任务完成

比如：Claude Code、Codex、Gemini CLI
构建方案比如：React

## Skill(Agent Skill)
>Skills 本质上就是教 AI 按固定流程做事的操作说明书，一旦写好，就能像函数一样反复调用。

我们可以把 Skills 看成把 某类事情应该怎么专业做 这件事，封装成一个可复用、可自动触发的能力模块。
因为Skill和MCP有点像，做个和MCP的类比：MCP 是专业厨房，提供了各种工具、食材和设备；Skill 是菜谱，告诉你怎么用这些东西做出一道菜。
