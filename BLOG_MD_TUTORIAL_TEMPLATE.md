# Blog Markdown 书写教程模板

这是一份面向本项目的 Markdown 写作模板，包含常用语法与可直接复用的文章骨架。

## 1. 快速开始

1. 在写作页输入标题和 slug。
2. 正文按下面模板编写。
3. 预览确认后发布。

## 2. 常用语法速查

### 2.1 标题与目录

```md
# 一级标题
## 二级标题
### 三级标题
```

建议：正文结构尽量使用到三级标题，目录展示更清晰。

### 2.2 文本样式

```md
这是普通文本。

**这是加粗**
*这是斜体*
~~这是删除线~~
`这是行内代码`
```

### 2.3 列表

```md
- 无序项 A
- 无序项 B
  - 子项 B.1

1. 有序项 1
2. 有序项 2
```

任务列表：

```md
- [ ] 待完成事项
- [x] 已完成事项
```

### 2.4 链接与图片

链接：

```md
[我的主页](https://example.com)
```

网络图片：

```md
![](https://example.com/demo.png)
```

本项目本地图片占位（由编辑器自动生成，发布时会替换）：

```md
![](local-image:your-image-id)
```

### 2.5 引用与分割线

```md
> 这是一段引用。

---
```

### 2.6 代码块（支持高亮）

```md
```ts
export function sum(a: number, b: number) {
  return a + b
}
```
```

可替换为 `js`、`ts`、`tsx`、`bash`、`json` 等语言标签。

### 2.7 数学公式（支持 KaTeX）

行内公式：

```md
质能方程：$E = mc^2$
```

块级公式：

```md
$$
\int_0^1 x^2 dx = \frac{1}{3}
$$
```

### 2.8 表格

```md
| 字段 | 说明 |
| --- | --- |
| title | 文章标题 |
| slug | 文章路径 |
| summary | 文章摘要 |
```

## 3. 可直接复制的文章骨架

把下面内容整体复制到编辑器后再改：

```md
# 文章标题

> 一句话导读：这篇文章解决了什么问题。

## 背景

这里写背景信息和使用场景。

## 目标

- 目标 1
- 目标 2
- 目标 3

## 实现步骤

### 步骤一：准备

说明准备内容。

### 步骤二：编码

示例代码：

```ts
type User = {
  id: string
  name: string
}

const users: User[] = [{ id: 'u1', name: 'Alice' }]
```

### 步骤三：验证

- [ ] 自测主要功能
- [ ] 覆盖边界场景
- [ ] 完成发布前检查

## 常见问题

### 问题 1

问题描述与解决方案。

### 问题 2

问题描述与解决方案。

## 总结

本文完成了：

1. 结果 A
2. 结果 B
3. 后续优化方向

## 参考链接

- [官方文档 1](https://example.com)
- [官方文档 2](https://example.com)
```

## 4. 写作建议

1. 一段只表达一个重点，尽量短句。
2. 标题要像目录，不要模糊命名。
3. 代码块尽量标注语言。
4. 发布前检查图片链接和公式渲染。
5. 摘要建议 1 到 2 句话，突出结论。
